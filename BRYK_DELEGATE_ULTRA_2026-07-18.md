# BRYK — Ультра-делегат: параллельная волна

**Дата:** 2026-07-18 · **Кому:** Opus в терминале (оркестратор) · **От:** Fable (Cowork)
**Контекст:** канон = `BRYK_MASTER_PLAN_2026-07-18.md` (§17 — сверка и логика панели; R10/R11 — ресёрч эффектов; §10–§16 — твои кодовые секции). База реализована, гейты app 55/55.
**Режим:** максимум параллельных субагентов. Схема §17.3: ты = ИНТЕГРАТОР (единственный владелец `index.html`), воркеры владеют своими файлами, сдают модуль + анкер-спеку (≤30 строк) + контрактный гейт.

## 0. Перед стартом (сам, до спавна воркеров)

1. **`git init` в `bryk/`** + коммит текущего состояния. Без этого не стартовать (урок MUSICVIZ/rec-label).
2. Прогнать гейты (core · panel · app) — зафиксировать зелёную базу.
3. Спавнить воркеров по трекам ниже. Каждому в промпт: его трек целиком + пути исходников + правило «чужие файлы не трогать».

## 1. Железные правила волны (нарушение = откат)

- Один владелец файла. `index.html` правит ТОЛЬКО интегратор, анкерными патчами.
- Запись в модулируемые параметры — только через `setDriven`/базы (§15.5).
- Гейтить контракт, не инвентарь (§16.4). Гейт после КАЖДОГО мержа, коммит после каждого зелёного гейта.
- Решения Роба §10.0 в силе: тёмная база · палитра диапазоном · панели по грамматике Synthex (зоны — §17.1) · пост-эффекты отдельным слотом, не на ходу · сферу/шар не делать.
- Канон-док только дописывается датированными секциями. Скрин перед сдачей каждого визуального изменения.

## 2. Треки воркеров

### Очередь ИНТЕГРАТОРА (index.html, последовательно, в этом порядке)

**I1 · MACRO** — мастер-макро ENERGY/DENSITY/CHAOS (§9.2, §17.1). Три скраба верх левой панели; раскладка макро→параметры декларацией (черновик готовит W9), живёт в сцене v+1 (миграция по §15.3). Через базы. Гейт: макро двигает ≥5 параметров каждый; сцена сохраняет/восстанавливает.
**I2a · SLOTS-EQJ** 🔴 *(переписано 2026-07-19 по живому фидбеку — спека A3.1, ПРИОРИТЕТ №1)* — weight-blend слоёв ОТМЕНЁН: слот S1–S4 = честный слой по модели EQJ: **свой суб-пул тел (партиция), свой blend (normal/multiply/screen/difference/exclusion), своя независимая opacity**. Все 4 живут одновременно, не переключаются. Силы блока — только своим телам; межслойно — только avoid/collide через общий grid. Банк блоков → назначение в слот (модель MixEmergency SAMPLES→BANK). Пустой слот = empty state. Сцены v+1 (миграция со статусом). Клавиатура A2.3. Гейт: 4 разных картинки различимы (скрин), multiply/screen реально работают, поведение одного слоя не двигает чужие тела.
**I2c · LEAD** *(A3.2)* — лидер-слой: эмфаза энергии/яркости лидера ~1.0, остальные ~0.5; клавиши 1–4 переводят лидерство кроссфейдом по ease Motion Pad 0.6–1.2с, квантуемо к bar. needs-Rob: финальная семантика после первого прогона.
**I2d · MASTER + CAL-SENS** 🔴 *(A3.4–A3.5)* — мастер-фейдер стека (MIX) + калибровочный наряд ПО ВСЕМ параметрам: диапазон = рабочая зона (экстрим только typed/allowExtend; «нужно 0.5 → слайдер до ~2, не до 500»), нейтральный дефолт каждого блока = спокойный малевич-танец (A3.3: avoid по дефолту, log-размеры крупные+мелкие, редкие инверсии, равновесие), exp-кривые на скоростных ручках, мод-глубины дефолт ≤0.15, мягче atk/rel. Сдача: таблица калибровки (param·range·default·curve) секцией в канон. Гейт: при дефолтах ни один параметр не «дёргается» (DRIVEN_LOG variance-порог).
**I2 · PANEL-ZONES** — финализация зон панели по карте §17.1 (MACRO·MODES·LOOK·WORD | INPUT·CLOCK·RACK·SCENES); зона MODES = 4 карточки слотов из I2a. Не редизайн — раскладка. Гейт: panel-гейт + скрины обеих колонок.
**I3 · FX-CANVAS** — эхо-туннель (feedback: кадр в себя, zoom·rotate·decay; мишени zoom←bassSlow, rotate←beat.phase) + волна-осциллограф (time-domain кольцом/лентой/контуром фигуры) + калейдоскоп N∈{2,4,6,8}. Математику/пресеты готовит W5. Гейт: FPS ≥55 при всех трёх включённых.
**I4 · SCENE-ASSETS** — сериализация ассетов в сцену: файлы в `bryk/assets/` по ссылке + base64-фолбэк <200КБ (решение зафиксировать в канон-доке). Камера-маска+ассет вместе (сейчас либо/либо, §15.6).
**I5 · MERGE-ПРИЁМ** модулей воркеров W3–W8 по мере сдачи (анкер-спеки).

### Параллельные воркеры (свои файлы, стартуют разом)

**W3 · AUDIO-ROBUST** — владеет `audio-core.js` + `audio-core.test.js`. По §17.2: CAL noise-floor (API захвата фона + вычитание по полосам) · band-селектор бита (sub/bass) · refractory ~100мс + гистерезис в onset · adaptive whitening (Stowell & Plumbley) · rmsSlow-gate покоя · confidence-гейт BPM. Контракт `Audio.features` НЕ ломать, только расширять. Гейт: node-тест BPM ±2 зелёный + новый тест «фикстура+белый шум −10дБ: onset false-rate <5%». Анкер-спека: кнопка CAL + селектор в INPUT.
**W4 · QUAD-WAVE** — владеет `probe/quad-wave.cjs`. Деформация сетки drawQuadWorld (R10.3): displacement-колбэк на узлы 5×5 → FLAG-волна/MORISAWA-грид на любой текстуре. Разработка на копии функции в probe, гейт: узлы конечны, волна периодична, нулевой displacement = битэквивалент старому. Сдаёт патч-спеку (сигнатура + 2 пресета волны + мишень amp←midSlow).
**W5 · FX-MATH** — владеет `probe/fx.cjs`. Математика/пресеты для I3: матрицы калейдоскоп-секторов, кривые decay эха, раскладки волны (кольцо/лента/контур). Headless-проверки, сдаёт интегратору.
**W6 · FLUID** — владеет `fluid-layer.js`. Адаптация PavelDoGreat script.js (MIT, dat.gui выкинуть): API `initFluid(canvas)` · `splat(x,y,dx,dy,color)` · `fluidStep(features)` (splat←onset в позициях тел, сила←bassFast, dye←палитра, vorticity←midSlow). Слой в стопку под #geo со своим PRESENCE (паттерн обвязки — §10.2: отдельный канвас, ленивый init, тихий glFail). Гейт: standalone-страница `probe/fluid.html` рендерит, деградация без WebGL чистая.
**W7 · MILK** — владеет `milk-layer.js` + `milk-presets.js`. butterchurn (MIT) одним бандлом без npm-рантайма; ест наш AnalyserNode; API `initMilk(canvas, analyser)` · `milkPreset(i|name, blendSec)` · curated-набор 12–20 пресетов из butterchurn-presets (тёмные, графичные — под палитру BRYK). Гейт: standalone `probe/milk.html`.
**W8 · RD** — владеет `rd-layer.js`. Свой Gray-Scott по спеке R10.1 (~30 строк GLSL, ping-pong): k/f-дрейф←midSlow · seed-пятно←onset · color map из палитры · Flow/Orientation-поля (Radial/Swirl/Vortex минимум). Стартует ПОСЛЕ сдачи W6 (переиспользует паттерн слоя). Гейт: standalone `probe/rd.html`.
**W9 · DATA** — владеет `shapes-data.js` + `macro-map.js`. Полная таблица form-полиэдров из particle-dance (§16.5) · черновик раскладки макро→параметры для I1 (по 8–12 мишеней на макро с весами) · чистка приоритета манеры/поведения (предложение: манера = оверлей ПОВЕРХ поведения, три поля flow/cohesion/separation — только от манеры; спека, применяет интегратор).
**W10 · QA-EYES** — владеет `probe/*` QA-скриптами + скрин-лупом. Расширить контрактные гейты (каждый новый модуль = контракт, не счёт) · eyes-луп: скрины light/dark/перформанс-режим ×3 состояния сцены · FPS-бюджет-гейт (сцена из 3 режимов + камера + эхо ≥55fps) · WCAG-проход панелей (контраст, focus-visible) · **A1.3-1: FPS/chroma-бенч в app.cjs обернуть try/catch и печатать итог ВСЕГДА, бенч после summary** (на софт-WebGL бенч убивает браузер до итоговой строки — ловлено во внешнем аудите A1).

## 3. Порядок и зависимости

```
 старт:  git init → гейты → спавн W3 W4 W5 W6 W7 W9 W10 (разом)
 I2a SLOTS-EQJ 🔴 → I2d MASTER+CAL-SENS 🔴 → I2c LEAD → I1 MACRO (ждёт W9) → I2 PANEL → I3 FX (ждёт W5) → I4 SCENE-ASSETS
 (приоритет очереди перевёрнут 2026-07-19: живой фидбек Роба A3 первым — слои и чувствительность, макро после)
 W8 RD — после сдачи W6
 мержи I5 — по мере готовности, каждый со своим гейтом и коммитом
```

Приоритет при нехватке рук: **I1 → I2 → W3 → I3 → W6 → W7 → остальное** (макро и панель — просьба Роба №1; аудио-надёжность — практики туса; эффекты следом).

## 4. Что НЕ делать в этой волне

Пост-эффекты «на ходу» к экспериментам (Роб §10.0.6) · сфера/шар · редизайн панелей · WebGL в пуле тел (только слои-модули) · приёмы-редактор из UI (следующая волна, по глазу Роба) · пороги LOD/QUAD_BUDGET не крутить без Роба (§15.6 «открыто по глазу»).
